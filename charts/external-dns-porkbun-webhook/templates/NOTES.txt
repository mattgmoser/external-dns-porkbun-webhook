WARNING: THIS STANDALONE CHART IS DEPRECATED.

The ExternalDNS webhook protocol has no authentication. This release exposes
the DNS mutation endpoint through a ClusterIP Service. Prefer the official
ExternalDNS sidecar installation documented in the project README, where the
provider endpoint is bound to 127.0.0.1.

{{ include "edns-porkbun.fullname" . }} v{{ .Chart.AppVersion }} installed in legacy standalone mode.

Webhook URL (point external-dns at this from inside the cluster):
  http://{{ include "edns-porkbun.fullname" . }}.{{ .Release.Namespace }}.svc.cluster.local:{{ .Values.service.webhookPort }}

An existing ExternalDNS deployment using this legacy topology must pass:

  --provider=webhook
  --webhook-provider-url=http://{{ include "edns-porkbun.fullname" . }}.{{ .Release.Namespace }}.svc.cluster.local:{{ .Values.service.webhookPort }}

The current official ExternalDNS chart creates its own webhook sidecar and does
not use a provider.webhook.url value. Migrate to the canonical same-Pod values
instead of using this Service for a new installation.
