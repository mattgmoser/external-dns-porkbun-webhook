external-dns-porkbun-webhook {{ .Chart.AppVersion }} is installed through the
official ExternalDNS 1.21.1 chart.

The ExternalDNS controller and Porkbun webhook share one Pod. The provider API
listens only on 127.0.0.1:8888; the Service exposes the separate ops endpoint.

Check both containers:
  kubectl -n {{ .Release.Namespace }} get pods
  kubectl -n {{ .Release.Namespace }} logs -l app.kubernetes.io/instance={{ .Release.Name }} -c external-dns
  kubectl -n {{ .Release.Namespace }} logs -l app.kubernetes.io/instance={{ .Release.Name }} -c webhook

Keep external-dns.txtOwnerId and external-dns.txtPrefix stable after records
have been created. Restart the Deployment after rotating Porkbun credentials.
